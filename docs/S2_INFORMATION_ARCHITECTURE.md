# S2 信息架构文档

**版本**: S2重构版  
**日期**: 2026-02-05

本文档描述S2重构后的信息架构设计。

---

## 1. 首页信息架构

### 1.1 首屏（Above the Fold）

**目标**: 零学习成本，3秒内理解价值

**内容层级**:
1. **标题**（H1）
   - "将课程大纲转化为结构化协作学习活动"
   - 一句话价值主张

2. **双角色入口**（等权卡片）
   - 卡片1: 登录为教师
   - 卡片2: 登录为学生
   - 并列显示，视觉权重相同

3. **Demo入口**（次级按钮）
   - "快速体验 Demo"
   - 独立于角色入口

### 1.2 可展开区域

**技术细节**（默认折叠）
- 多阶段生成
- 文档检索
- 调整记录
- 质量检查

**交互**: 点击"了解更多技术细节"展开/折叠

## 2. 教师端信息架构

### 2.1 顶部结构

```
[页面标题: 教师工作台] [主按钮: 新建活动]
[状态提示: 你现在在第 X/4 步]
```

### 2.2 主流程（4步卡片）

**布局**: 网格布局，响应式（移动端单列）

**每步卡片包含**:
- 步骤编号（1-4）
- 图标
- 标题
- 这一步做什么（1句话）
- 为什么要做（1句话）
- 完成后会得到什么（1句话）
- CTA按钮

**交互**: 
- 卡片可点击，跳转到对应步骤
- 当前步骤高亮
- 已完成步骤标记

### 2.3 技术详情抽屉（折叠）

**内容**:
- 活动项目统计
- 自动生成过程统计
- 待发布活动统计
- 质量检查结果统计

**交互**: 点击"查看技术详情"展开/折叠

## 3. 学生端信息架构

### 3.1 首屏信息（必须显示）

**当前活动卡片**
- 活动名称（H2）
- 截止时间（如有）

**本次任务卡片**
- 任务列表（最多3条）
- 任务描述
- 任务说明

**操作按钮**
- 提交任务（主按钮）
- 继续任务（次按钮）

### 3.2 可折叠区域

**评分标准摘要**
- 默认折叠
- 点击展开查看详细标准

**历史记录**
- 默认折叠
- 点击展开查看已完成活动

**协作建议**
- 默认折叠
- 点击展开查看协作技巧

---

## 4. 信息层级原则

### 4.1 优先级

1. **P0（必须首屏显示）**
   - 当前任务
   - 主要操作按钮
   - 关键状态信息

2. **P1（可折叠显示）**
   - 历史记录
   - 详细说明
   - 辅助信息

3. **P2（技术详情）**
   - 统计数据
   - 技术术语
   - 高级功能

### 4.2 认知负荷控制

- **首屏文本**: < 2000字符
- **主按钮数量**: <= 2
- **技术术语**: 不在首屏出现
- **步骤数量**: 4步（符合Miller's Law）

## 5. 导航结构

### 5.1 教师端侧边栏

```
教师工作台（Dashboard）
├── 活动项目
├── 教学目标检查
├── 自动生成过程
├── 课程文档
├── 教师调整记录
├── 质量检查结果
└── 发布与导出
```

### 5.2 学生端顶部导航

```
[Logo] [学生工作台] [用户信息]
```

**简化设计**: 学生端无需复杂导航，聚焦当前任务

## 6. 空状态设计

### 6.1 三要素原则

每个空状态必须包含：
1. **状态描述**: "当前没有活动"
2. **原因说明**: "URL中未提供活动ID，或教师尚未发布活动"
3. **下一步行动**: "请联系你的教师创建并发布活动"

### 6.2 视觉设计

- 图标（大尺寸，低饱和度）
- 标题（H4）
- 说明文本（2-3句）
- 操作按钮（如有）

---

## 7. 响应式设计

### 7.1 断点

- **移动端**: < 768px
- **平板**: 768px - 1024px
- **桌面**: > 1024px

### 7.2 布局调整

- **首页**: 移动端单列，桌面端双列
- **教师端4步卡片**: 移动端单列，桌面端2-4列
- **学生端**: 移动端垂直堆叠，桌面端保持布局

---

## 8. 可访问性

### 8.1 ARIA标签

- 所有按钮包含`aria-label`
- 折叠区域使用`aria-expanded`
- 步骤指示使用`aria-current`

### 8.2 键盘导航

- Tab顺序符合视觉顺序
- Enter/Space激活按钮
- Escape关闭模态框
