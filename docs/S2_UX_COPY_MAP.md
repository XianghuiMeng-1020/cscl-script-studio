# S2 UX 文案映射表

**版本**: S2重构版  
**日期**: 2026-02-05

本文档记录S2重构中的术语替换，确保面向用户的文案清晰易懂。

---

## 核心术语映射

| 旧术语（技术化） | 新术语（用户友好） | 使用场景 |
|----------------|------------------|---------|
| Script Project | 活动项目 | 教师端导航、卡片标题 |
| Spec Validation | 教学目标检查 | 教师端导航、步骤名称 |
| Pipeline Run | 自动生成过程 | 教师端导航、状态显示 |
| Quality Report | 质量检查结果 | 教师端导航、报告标题 |
| Decision Timeline | 教师调整记录 | 教师端导航、历史记录 |
| Script | 活动方案 / 活动流程 | 通用，根据上下文选择 |
| Pipeline | 自动生成过程 | 仅在技术详情中保留 |
| RAG | 文档检索 | 仅在技术详情中保留 |
| Refiner | 优化阶段 | 仅在技术详情中保留 |

---

## 按钮文案规范

### 动作动词 + 结果预期

| 功能 | 按钮文案 | 说明 |
|------|---------|------|
| 创建新活动 | "新建活动" | 动词开头，明确结果 |
| 导入大纲 | "开始导入" | 动作+预期 |
| 确认目标 | "确认目标" | 简洁明确 |
| 生成活动 | "开始生成" | 动作+预期 |
| 发布活动 | "审阅发布" | 两步合并，清晰流程 |
| 提交任务 | "提交任务" | 学生端，直接明确 |
| 继续任务 | "继续任务" | 学生端，动作明确 |

---

## 页面标题规范

| 页面 | 标题 | 说明 |
|------|------|------|
| 首页 | "将课程大纲转化为结构化协作学习活动" | 一句话价值主张 |
| 教师端 | "教师工作台" | 角色+功能 |
| 学生端 | "学生工作台" | 角色+功能 |

---

## 状态提示规范

| 场景 | 文案 | 说明 |
|------|------|------|
| 步骤指示 | "你现在在第 X/4 步" | 清晰进度 |
| 当前状态 | "准备开始" / "导入课程大纲" / ... | 当前步骤名称 |
| 空状态 | "当前没有活动" + 原因 + 下一步 | 三要素：状态+原因+行动 |

---

## 错误消息规范

### 格式：问题 + 原因 + 解决方案

| 错误类型 | 示例 | 说明 |
|---------|------|------|
| 文件类型不支持 | "PDF解析暂未支持。请上传 .txt 或 .md 文件，或先将PDF内容复制为文本后粘贴上传。" | 明确问题+替代方案 |
| 编码错误 | "无法解码文件，尝试的编码：utf-8, gbk, big5...。请确保文件为UTF-8、GBK或Big5编码。" | 列出尝试的编码+建议 |
| 文本过短 | "文本内容过短（至少10个字符）。请提供有效的课程大纲内容。" | 明确要求+期望内容 |
| 权限错误 | "当前角色无权限查看此活动。" | 简洁明确，不暴露技术细节 |

## 首屏文案限制

### 禁止在首屏出现的术语
- Pipeline
- RAG
- Refiner
- Planner
- Critic
- Material Generator
- Token
- Embedding
- Vector

### 允许在"技术详情"中使用的术语
- 所有技术术语可在折叠区域使用
- 需提供简短解释

---

## 多语言支持

### 当前支持语言
- 简体中文（主要）
- English（部分）
- 繁體中文（部分）

### 文案原则
- 避免术语堆叠
- 使用日常用语
- 提供上下文说明
