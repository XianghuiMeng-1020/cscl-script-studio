# S2.2 PDF 文本提取说明文档

**版本**: 1.0.0  
**更新日期**: 2026-02-05

---

## 概述

本文档说明 PDF 文本提取的实现策略、失败处理机制和错误码定义。

---

## 实现策略

### 1. 解析库选择

使用 `pypdf` (>=3.17.0) 作为 PDF 解析库：
- 纯 Python 实现，无需外部依赖
- 支持文本提取和元数据读取
- 兼容多种 PDF 版本

### 2. 提取流程

```
PDF文件上传
    ↓
读取文件字节流
    ↓
使用 pypdf.PdfReader 解析
    ↓
逐页提取文本 (extract_text())
    ↓
合并所有页面文本
    ↓
normalize_text() 规范化处理
    ↓
验证最小长度 (≥80字符)
    ↓
返回提取的文本
```

### 3. 文本规范化 (normalize_text)

规范化步骤包括：

1. **Unicode 规范化**: NFKC 形式
2. **控制字符移除**: 保留换行符和制表符
3. **空白压缩**: 
   - 多个空格/制表符 → 单个空格
   - 多个换行符 → 双换行符
4. **零宽字符移除**: `\u200b-\u200d`, `\ufeff`
5. **PDF 标记移除**: 
   - `%PDF-X.X` 头部标记
   - PDF 对象标记 (`obj ... obj`)

### 4. 编码处理

对于文本文件 (TXT/MD)，支持以下编码（按优先级）：
1. UTF-8
2. UTF-8-sig (带 BOM)
3. GB18030 (简体中文)
4. Big5 (繁體中文)
5. Latin-1 (fallback)

## 失败处理

### 错误类型与处理

| 错误类型 | 错误码 | HTTP状态码 | 处理策略 |
|---------|--------|-----------|---------|
| PDF 解析失败 | `PDF_PARSE_FAILED` | 422 | 返回用户友好错误信息，建议检查 PDF 是否损坏或加密 |
| 不支持的文件类型 | `UNSUPPORTED_FILE_TYPE` | 415 | 明确告知支持的文件类型 |
| 文本过短 | `TEXT_TOO_SHORT` | 422 | 要求至少 80 字符 |
| 提取文本为空 | `EMPTY_EXTRACTED_TEXT` | 422 | 可能是扫描版 PDF 或加密 PDF |
| 其他提取错误 | `EXTRACTION_FAILED` | 400 | 通用错误处理 |

### 错误信息示例

**PDF 解析失败**:
```
PDF解析失败：无法提取文本。可能是扫描版PDF或加密PDF。
```

**文本过短**:
```
提取的文本过短（少于80字符）。PDF可能不包含可提取的文本内容。
```

**不支持的文件类型**:
```
不支持的文件类型：application/unknown。支持的类型：.txt, .md, .pdf
```

## 限制与约束

1. **文件大小**: 最大 10MB
2. **最小文本长度**: 80 字符
3. **支持的 PDF 版本**: PDF 1.0 - 2.0（pypdf 支持的范围）
4. **不支持的功能**:
   - 扫描版 PDF（图片 OCR）
   - 加密 PDF（需要密码）
   - 表单字段提取
   - 图片提取

---

## 测试用例

### 成功场景

1. **标准 PDF**: 包含可提取文本的 PDF
2. **多页 PDF**: 包含多页文本的 PDF
3. **中文 PDF**: 包含中文字符的 PDF

### 失败场景

1. **扫描版 PDF**: 仅包含图片的 PDF
2. **加密 PDF**: 需要密码的 PDF
3. **损坏的 PDF**: 文件结构损坏
4. **空 PDF**: 不包含任何内容
5. **文本过短**: 提取的文本少于 80 字符
